# 调研型（survey）实验设计模板

> 写作风格：用自然语言写清“怎么控变量、怎么跑流程、怎么评估”。避免模块清单与参数表堆砌，但必须保留清晰的指标口径与可复现输出说明。

## 1. 基本信息

本实验名称：【英文下划线格式，例如 info_propagation】

本实验目标：【一句话写清要比较的机制或干预效果，例如“比较不同干预条件对认知/态度/行为选择的影响”】【要求：明确是 survey 类实验】

背景设定：【2–4 句话说明人群/群体、信息或刺激的来源（政府/媒体/组织/系统提示等），以及实验发生的场景边界。必要时说明是否存在社交互动/网络扩散（没有也完全可以）。】

实验窗口与步进：【说明每个“处理条件/实验组”运行多少轮（total_years），一轮的时间单位（天/轮次/年/回合），以及不同条件之间如何重置状态与保证可比性。】

原需求（尽可能保留）：
【粘贴用户原始需求，保留策略枚举、提示语、公式、评估方法等关键细节。】

## 2. 核心对象与实验设定（自然语言口径）

参与者/智能体主体：【说明数量级、会做的事（接收刺激/信息、可能的互动、记忆更新、填写问卷/做选择），以及哪些状态会被重置以保证条件间可比性。】

互动与网络（可选）：【若实验包含社交互动/网络传播，用 1 段话说明网络如何影响扩散或同伴效应；若存在“种子/关键节点”选取，请说明依据（例如按中心度排序）。不包含网络时，本段可删除。】

刺激/信息与提示（可选）：【说明刺激材料来自哪里，是否有不同版本（例如短/长、强/弱、正/负框架），以及“额外提示/元信息”是否会改变参与者对他人认知或规则的理解。这里不要写成配置字段清单，只要讲清“变了什么、为什么”。】

## 3. 实验控制逻辑（必须写清“对照组是什么”）

用自然语言说明实验如何严格控制变量：同一批参与者、同一套环境与评分方式，仅改变少数“实验因素/处理条件”。强调哪些是被操控的自变量，哪些是被观察的因变量。

处理条件/实验组定义：【列出 2–6 个条件即可。每个条件用 1–2 句说明“参与者看到/经历了什么差异”。如果是二维或多维设计（A×B），说明每一维是什么，以及组合成哪些条件。】

若有公式（可选）：【直接写在本节，例如某种评分/选择规则/抽样概率；给出符号含义即可，不要展开推导。】

## 4. 实验流程（例：条件循环）

用 1 段话描述整体流程，然后用尽量少的句子写清每个条件内部的三段：

1）重置状态：重置时间与参与者的实验状态（记忆、计数器等），保证从同一初始条件出发。

2）时间步循环：每轮执行刺激呈现/干预 → 参与者反应（可包含互动/传播）→ 记忆总结/状态更新。

3）条件后评估：统一进行问卷/测试/选择题（可多份），并保存该条件的汇总结果。

## 5. 关键指标（必须保留）

本实验的主要评估单位通常是“处理条件/实验组”（而不是单个时间步），因此建议至少按条件汇总；如需要过程解释，可额外保留每轮中间数据：

- 过程指标（Process）：衡量实验运行过程中发生了什么（例如互动次数、传播量、响应率、覆盖率、完成率、停留时间等）。
- 认知指标（Knowledge/Understanding）：衡量理解/记忆/准确率（例如总体准确率、每题准确率、关键知识点掌握率、错误类型分布）。
- 行为/态度指标（Choice/Attitude）：衡量选择分布与倾向（例如 A/B 选择比例、风险偏好、信任/支持度量表分数、行动意愿等）。

如需更细解释，可补充“结构性指标”（例如网络中心节点贡献、传播深度、观点一致性/极化程度），但不要为了完整而强行加入。
