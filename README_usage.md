## 编码流程优化

### CodeArchitect 代码生成流程

**Simulator 代码生成（3步）：**
```
步骤1: 生成代码框架
  ↓
步骤2: 创建工作列表（识别不完整的函数）
  ↓
步骤3.1: 创建工作列表
  ├─ 识别不完整的函数（pass/TODO）
  ├─ 📋 输出详细工作列表（函数名、描述、所需模块）
  └─ 统计待办任务数量
  ↓
步骤3.2: 逐个补完函数
  ├─ 针对每个TODO/pass单独实现
  ├─ 显示进度 [1/N]、[2/N]...
  └─ 实时反馈成功/失败
步骤3.3: 总体检查与修复 ⭐合并优化
  ├─ 第1次检查（自动修复）
  │   ├─ 语法检查（async async def、缩进等）
  │   ├─ 导入检查（from .simulator_imports import *）
  │   ├─ 实现完整性（无pass/TODO）
  │   ├─ 代码质量（逻辑、异常处理）
  │   └─ ✓ 自动修复所有问题
  ├─ 第2次检查（验证修复）
  │   └─ 确认所有问题已解决
  └─ 结果：✓ 完整无误 或 ⚠️ 部分问题未解决
```

**Main 代码生成（3步）：**
```
步骤1: 生成入口文件框架
  ↓
步骤2: 创建工作列表（识别不完整的部分）
  ↓
步骤3: 逐个补完任务（对象初始化、配置加载等）
  ↓
步骤4.1: 创建工作列表
  ├─ 识别不完整的部分（TODO/pass）
  ├─ 📋 输出详细工作列表（任务名、描述、所需模块）
  └─ 统计待办任务数量
  ↓
步骤4.2: 逐个补完任务
  ├─ 对象初始化、配置加载等
  ├─ 显示进度 [1/N]、[2/N]...
  └─ 实时反馈成功/失败
  ↓
步骤4.3: 总体检查与修复 ⭐合并优化
  ├─ 第1次检查（自动修复）
  │   ├─ 缺失导入（argparse, os, sys, yaml）
  │   ├─ 未完成实现（TODO、pass）
  │   ├─ 对象初始化完整性
  │   ├─ 异常处理
  │   └─ ✓ 自动修复所有问题
  ├─ 第2次检查（验证修复）
  │   └─ 确认所有问题已解决
  └─ 结果：✓ 完整无误 或 ⚠️ 部分问题未解决
  ↓
步骤5: 总体检查 ⭐加强
```

### 改进亮点

✅ **1. 工作列表可视化**
```
📋 工作列表（共 3 项）:
  1. update_state
     描述: 更新系统状态（人口、气候等）
     模块: population, climate
  2. execute_actions
     描述: 执行居民决策和行为
     模块: resident, social_network
  3. collect_results
     描述: 收集本轮模拟结果
     模块: population
```

✅ **2. 检查与修复合并**
- 从4步简化为3步（步骤3.3和3.4合并）
- 一次性检查所有问题并自动修复
- 最多重试2次确保代码质量
- 避免"发现问题但不修复"的情况

✅ **3. 智能修复机制**
- LLM检查时如发现问题，直接返回修复后代码
- 如果第一次未修复，自动进行第二次修复尝试
- 确保问题得到有效解决

## 运行模式

系统提供两种运行模式：

### 1. 自动模式（选项 1）

**特点：**
- ✅ 一次性完成所有步骤
- ✅ 无需等待，自动生成所有内容
- ✅ 适合快速原型开发

**流程：**
```
输入需求 → 设计 → 编码 → 完成
```

### 2. 交互模式（选项 2）

**特点：**
- ✅ 每个阶段完成后等待用户反馈
- ✅ 可以重新生成不满意的部分
- ✅ 支持版本控制和迭代优化

**流程：**
```
输入需求 
  ↓
阶段1: 项目初始化
  ↓
阶段2: 系统设计
  ├─ 显示结果（需求、模块、设计文档）
  ├─ 等待反馈
  │   - 'ok' → 继续
  │   - 反馈意见 → 重新生成（带上一版本+反馈）
  │   - 'quit' → 退出
  ↓
阶段3: 代码生成
  ├─ 显示结果（simulator、main、配置、提示词）
  ├─ 等待反馈
  │   - 'ok' → 完成
  │   - 反馈意见 → 重新生成（带上一版本+反馈）
  │   - 'quit' → 退出
  ↓
完成
```

## 建议

- **首次使用** → 选择交互模式，了解每个阶段的输出
- **快速测试** → 选择自动模式
- **重要项目** → 选择交互模式，仔细审查每个阶段
- **模板生成** → 自动模式生成初稿，然后手动微调

## 常见问题

**Q: 交互模式可以返回上一阶段吗？**  
A: 当前版本不支持，但可以在当前阶段重新生成多次

**Q: 如何保存历史版本？**  
A: 交互模式会自动在内存中保存所有版本，日志文件也会记录

**Q: 反馈意见有格式要求吗？**  
A: 没有，用自然语言描述即可，如"增加XX模块"、"调整XX参数"

**Q: 可以中途退出吗？**  
A: 可以，输入 'quit' 即可安全退出
