### 改进亮点

✅ **1. 工作列表可视化**
```
📋 工作列表（共 3 项）:
  1. update_state
     描述: 更新系统状态（人口、气候等）
     模块: population, climate
  2. execute_actions
     描述: 执行居民决策和行为
     模块: resident, social_network
  3. collect_results
     描述: 收集本轮模拟结果
     模块: population
```

✅ **2. 循环重试机制** ⭐ 新增
- **每个小步骤独立重试**
  - 创建工作列表：最多5次重试
  - 补完每个函数/任务：最多5次重试
  - 总体检查与修复：最多2次重试

- **智能失败诊断**
  - 超过5次失败 → 自动触发大模型诊断
  - 分析根本原因（代码结构/逻辑/格式）
  - 具体问题定位（行号/函数名）
  - 详细解决方案 + 预防措施

- **实时进度反馈**
  ```
  尝试 [1/5]: 补完函数 update_state...
  ⚠️ 第 1 次补完失败
  尝试 [2/5]: 补完函数 update_state...
  ✓ 成功补完函数: update_state
  ```

✅ **3. 检查与修复合并**
- 从4步简化为3步（步骤3.3和3.4合并）
- 一次性检查所有问题并自动修复
- 最多重试2次确保代码质量
- 避免"发现问题但不修复"的情况

✅ **4. 智能修复机制**
- LLM检查时如发现问题，直接返回修复后代码
- 如果第一次未修复，自动进行第二次修复尝试
- 确保问题得到有效解决
- 失败时提供详细诊断报告

## 运行模式

系统提供两种运行模式：

### 1. 自动模式（选项 1）

**特点：**
- ✅ 一次性完成所有步骤
- ✅ 无需等待，自动生成所有内容
- ✅ 适合快速原型开发

**流程：**
```
输入需求 → 设计 → 编码 → 完成
```

### 2. 交互模式（选项 2）

**特点：**
- ✅ 每个阶段完成后等待用户反馈
- ✅ 可以重新生成不满意的部分
- ✅ 支持版本控制和迭代优化

**流程：**
```
输入需求 
  ↓
阶段1: 项目初始化
  ↓
阶段2: 系统设计
  ├─ 显示结果（需求、模块、设计文档）
  ├─ 等待反馈
  │   - 'ok' → 继续
  │   - 反馈意见 → 重新生成（带上一版本+反馈）
  │   - 'quit' → 退出
  ↓
阶段3: 代码生成
  ├─ 显示结果（simulator、main、配置、提示词）
  ├─ 等待反馈
  │   - 'ok' → 完成
  │   - 反馈意见 → 重新生成（带上一版本+反馈）
  │   - 'quit' → 退出
  ↓
完成
```

## 建议

- **首次使用** → 选择交互模式，了解每个阶段的输出
- **快速测试** → 选择自动模式
- **重要项目** → 选择交互模式，仔细审查每个阶段
- **模板生成** → 自动模式生成初稿，然后手动微调

## 常见问题

**Q: 交互模式可以返回上一阶段吗？**  
A: 当前版本不支持，但可以在当前阶段重新生成多次

**Q: 如何保存历史版本？**  
A: 交互模式会自动在内存中保存所有版本，日志文件也会记录

**Q: 反馈意见有格式要求吗？**  
A: 没有，用自然语言描述即可，如"增加XX模块"、"调整XX参数"

**Q: 可以中途退出吗？**  
A: 可以，输入 'quit' 即可安全退出
