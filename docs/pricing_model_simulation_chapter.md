# 论文章节：基于多智能体的动态定价与库存管理模拟研究

## 1. 引言

在竞争激烈的市场环境中，动态定价与库存管理是企业维持盈利能力和市场竞争力的核心策略。价格不仅直接影响消费者需求和企业收入，还与库存水平、存储成本和缺货风险等供应链环节紧密相连。理解这些因素之间的复杂动态关系，对于制定最优的定价和库存策略至关重要。

本研究构建了一个基于多智能体（Multi-Agent System, MAS）的模拟模型，旨在探索一个由自主决策的消费者（智能体）构成的市场中，商品价格、库存水平和消费者满意度之间的动态演化过程。通过模拟大量智能体在市场中的购买和存储行为，我们旨在回答以下核心问题：
1.  市场均衡价格是如何形成的？其受到哪些关键因素（如需求弹性、财富水平）的影响？
2.  消费者的库存行为（存储、消耗）如何影响市场的整体库存水平和价格稳定性？
3.  如何量化和评估市场的宏观经济指标，如GDP、消费者满意度，以及它们与个体行为的关联？
4.  市场是否存在缺货风险？其概率如何演变？

本章节将详细阐述该模拟模型的设计、核心机制、模拟流程、关键参数以及用于评估的指标体系。

## 2. 模拟模型设计

为了研究上述问题，我们设计了一个包含消费者智能体、市场环境和宏观经济指标的计算模型。

### 2.1. 模型概述

本模型模拟了一个由众多消费者智能体组成的虚拟社会。这些智能体（`Resident`）具备自主决策能力，在每个模拟周期（年份）内，根据自身的财富、库存、价格敏感度以及当前市场价格等信息，做出购买、存储或调整策略的决策。模型的核心是观察和记录市场均衡价格、总库存水平、缺货概率等关键指标如何因智能体的集体行为而波动，并分析其背后的经济学原理。

### 2.2. 智能体与实体设计

#### 2.2.1. 消费者智能体 (Resident Agent)

消费者是本模型的核心决策单元，具备以下特征和行为：

-   **属性**: 每个智能体拥有财富（`wealth`）、商品库存（`inventory`）、价格敏感度（`price_sensitivity`）和满意度（`satisfaction`）等属性。
-   **核心行为**: 在每个模拟时间步，每个智能体利用大型语言模型（LLM）进行决策（`decide_action_by_llm`）。决策的上下文包括当前市场价格、自身财富与库存、存储成本、折旧率等。智能体的可选行动包括：
    -   **购买 (purchase)**: 根据当前价格和财富，决定购买一定数量的商品。
    -   **出售 (store/sell)**: 根据当前价格和库存，决定出售一定数量的商品以获取收入。
    -   **维持 (adjust_price/maintain)**: 保持当前状态，并可能根据库存水平微调未来的价格敏感度。
-   **动态演化**: 模拟引入了人口动态变化。系统会根据上一周期的平均满意度（`average_satisfaction`）调整出生率（`birth_rate`），并生成新的消费者智能体，模拟了市场参与者的内生增长。同时，智能体也存在生命周期，如果财富无法覆盖基本生活成本，则会“死亡”并被移出模拟。

#### 2.2.2. 市场环境 (Market Environment)

模型中没有明确的“交易所”实体，而是通过一个抽象的市场环境来体现价格机制。

-   **价格更新机制**: 市场的“均衡价格” (`equilibrium_price`) 并非由某个中心化机构设定，而是根据所有消费者的总需求和总供给动态计算得出。这反映了一个去中心化的市场价格形成过程。
-   **宏观经济指标**: 环境负责计算和记录宏观指标，如本地生产总值（`gdp`）和所有消费者的平均满意度（`average_satisfaction`）。

## 3. 核心机制与模拟流程

#### 3.1. 核心机制

1.  **市场均衡价格 (Equilibrium Price)**: 这是反映当前市场供需关系的核心指标。根据 `calculate_equilibrium_price` 函数，其计算方式基于供需比率和需求弹性。公式如下：
    $$ P_{eq} = P_{initial} \times \left( \frac{\sum Supply}{\sum Demand} \right)^{-\eta} $$
    其中，$P_{eq}$ 是均衡价格，$P_{initial}$ 是初始基准价格，$\sum Supply$ 是所有智能体的总库存量，$\sum Demand$ 是基于财富和价格敏感度估算的总需求量，$\eta$ 是需求价格弹性。

2.  **消费者LLM决策**: 每个智能体通过调用LLM，传入包含个人状态和市场信息的复杂上下文，来做出近似理性的经济决策。这种设计使得智能体的行为更具多样性和现实性，而非遵循固定的、简单的规则。

3.  **库存动态与成本**: 智能体的库存会因购买而增加，因出售而减少。持有库存会产生存储成本（`storage_cost_rate`）并面临商品折旧（`storage_depreciation`），这些成本会直接从智能体的财富中扣除。

#### 3.2. 模拟流程

模拟以离散的时间步（年份）进行，遵循 `simulator.run()` 方法中定义的循环。

1.  **初始化阶段**:
    -   根据 `config/pricing_model_simulation/simulation_config.yaml` 加载所有参数。
    -   创建地图、时间、人口等宏观环境对象。
    -   生成初始的消费者智能体群体。
    -   初始化模拟结果记录器。

2.  **迭代阶段 (按年份循环)**:
    -   **状态更新**: 计算当前周期的GDP和平均满意度，并基于满意度更新人口出生率。
    -   **人口增长**: 根据 `birth_rate` 生成新的消费者智能体，并将其整合进模拟系统。
    -   **市场价格计算**: 基于当前所有智能体的总供给（库存）和总需求（估算），计算出本周期的市场均衡价格。
    -   **智能体行动**:
        -   遍历所有智能体，检查并处理“死亡”事件。
        -   所有幸存的智能体并行执行其LLM决策。
        -   根据决策结果，处理每个智能体的购买或出售行为，更新其财富和库存。
    -   **库存成本结算**: 对所有持有库存的智能体，扣除存储成本和折旧。
    -   **数据收集**: 调用 `collect_results` 方法，记录当前年份的所有关键指标。
    -   **时间推进**: 时间进入下一年，循环继续，直至达到 `total_years`。

3.  **结束与分析阶段**:
    -   模拟循环结束后，将所有收集到的时间序列数据完整保存到 CSV 文件中。
    -   输出总模拟耗时。

## 4. 关键参数与可配置性

本模型具有高度的可配置性，研究者可通过修改位于 `config/pricing_model_simulation/` 目录下的 `simulation_config.yaml` 文件来调整实验条件。主要参数包括：

-   `initial_price`: 市场初始基准价格。
-   `demand_elasticity`: 需求价格弹性，影响价格对供需变化的敏感度。
-   `storage_cost_rate`: 库存的单位存储成本率。
-   `storage_depreciation`: 库存的折旧率。
-   `storage_capacity`: 每个智能体的最大库存容量。
-   `initial_population`: 初始市场参与者（消费者）的数量。
-   `birth_rate`: 基础人口年增长率。
-   `basic_living_cost`: 智能体维持生存所需的基础生活成本。

通过调整这些参数，可以系统性地研究不同市场结构和消费者行为模式对价格动态、库存管理和宏观经济的影响。

## 5. 数据收集与评估指标

为了全面评估市场的动态和效率，我们在每个模拟年份收集了以下一系列关键指标：

| 指标名称 (代码内) | 指标含义与计算方式 | 研究意义 |
| :--- | :--- | :--- |
| `equilibrium_price` | 市场均衡价格，根据总供需计算。 | 反映了市场的即时价格水平和供需关系，是核心观察变量。 |
| `inventory_level` | 所有消费者持有的总库存量。 | 衡量市场的整体供给水平和缓冲能力。 |
| `stockout_probability` | 库存为零或负的消费者占总人口的比例。 | 量化了市场面临的缺货风险，是评估市场稳定性的重要指标。 |
| `average_satisfaction` | 所有消费者的平均满意度。 | 作为衡量宏观社会福利的代理指标，并影响人口增长。 |
| `population` | 消费者总数。 | 跟踪市场规模的变化，可与其它指标关联分析规模效应。 |
| `gdp` | 本地生产总值，当前简化为所有消费者的总收入。 | 衡量市场经济活动的总产出。 |

## 6. 实验结果的体现与分析

模拟的输出结果为后续的深入分析提供了坚实基础：

1.  **原始数据**: 所有在模拟过程中按年份收集的指标数据，以时间序列的形式保存在 `data/` 目录下的 CSV 文件中（文件名格式为 `running_data_{timestamp}_pid{pid}.csv`）。这为精细的量化分析、统计检验和可视化提供了数据源。
2.  **可视化分析**: 虽然模拟器本身不直接生成图表，但产出的CSV数据可以被用于绘制各种分析图表，例如：
    -   **均衡价格走势图**: 绘制 `equilibrium_price` 的时间序列，观察价格的波动、趋势和收敛情况。
    -   **库存与缺货概率图**: 将 `inventory_level` 和 `stockout_probability` 绘制在同一时间轴上，分析库存周期和缺货风险的关联。
    -   **宏观经济指标图**: 绘制 `population`、`gdp` 和 `average_satisfaction` 的变化趋势，理解市场增长与社会福利的演变。

通过对这些结果的综合分析，研究者可以验证关于动态定价和库存管理的理论假设，识别影响市场稳定性的关键因素，并为制定更有效的市场策略提供数据支持和洞见。
