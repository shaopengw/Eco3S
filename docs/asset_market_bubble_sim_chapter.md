# 论文章节：基于多智能体的资产市场泡沫形成与演化模拟研究

## 1. 引言

资产泡沫（Asset Bubble）是金融市场中的一种普遍现象，指资产价格大幅偏离其基础价值（Fundamental Value）并持续上涨的状态。理解泡沫的形成机制、演化过程和崩溃条件，对于金融监管、风险管理和投资决策都具有重要意义。传统的金融理论基于理性预期假设，难以完全解释现实市场中频繁出现的泡沫现象。行为金融学的兴起为理解这一现象提供了新的视角，强调投资者的有限理性、异质性信念和社会学习在泡沫形成中的作用。

本研究构建了一个基于多智能体（Multi-Agent System, MAS）的实验资产市场模拟模型，旨在复现和探究资产泡沫的内生形成过程。该模型借鉴了Smith等人（1988）提出的经典实验资产市场范式，通过模拟大量具有异质性交易策略和风险偏好的智能体在有限期限内进行资产交易，观察市场价格偏离基础价值的动态过程。我们旨在回答以下核心问题：
1.  在智能体异质性和有限理性的条件下，资产价格是否会偏离其基础价值，形成泡沫？
2.  泡沫的形成和演化呈现怎样的时间模式？哪些因素影响泡沫的幅度和持续时间？
3.  市场的微观结构（如订单簿机制、买卖价差）如何影响价格发现和泡沫动态？
4.  智能体的交易行为模式（如追涨、理性套利）如何演变，并如何集体影响市场结果？

本章节将详细阐述该模拟模型的设计、核心机制、模拟流程、关键参数以及用于评估的指标体系。

## 2. 模拟模型设计

为了研究上述问题，我们设计了一个包含异质性交易者智能体、实验资产市场和宏观跟踪系统的计算模型。

### 2.1. 模型概述

本模型模拟了一个经典的实验资产市场环境。市场中交易一种虚拟资产，该资产具有明确的基础价值，由每期固定分红和剩余交易期数决定。众多交易者智能体（`Resident`）在有限的交易期内，通过一个连续双向拍卖（Continuous Double Auction）机制提交买卖订单，进行资产交易。智能体具备异质性的交易策略和风险偏好，并利用大型语言模型（LLM）进行决策，使其行为兼具理性分析和行为偏差的特征。模型的核心是观察和记录资产价格如何因智能体的集体交易行为而偏离基础价值，形成泡沫，并分析泡沫的动态演化路径。

### 2.2. 智能体与实体设计

#### 2.2.1. 交易者智能体 (Trader Agent)

交易者是本模型的核心决策单元，具备以下特征和行为：

-   **初始禀赋**: 每个智能体在模拟开始时被赋予相等数量的现金（`initial_cash`）和资产（`assets`）。这确保了初始条件的公平性，使得价格动态完全由交易行为驱动。
-   **异质性特征**: 每个智能体具有独特的交易策略（`trading_strategy`）和风险偏好（`risk_preference`）。交易策略包括"动量交易"（追涨杀跌）、"价值投资"（基于基础价值交易）和"混合策略"等。风险偏好包括"风险厌恶"、"风险中性"和"风险偏好"，这些特征影响智能体对价格波动和潜在收益的反应。
-   **核心行为**: 在每个交易期，智能体通过LLM进行决策（`resident_submit_order`）。决策的上下文包括：
    -   当前市场价格与资产基础价值
    -   订单簿中的买卖挂单数量
    -   资产的剩余分红期数和每期分红额
    -   自身持有的现金和资产数量
    -   价格的历史走势（上升或下降趋势）
    
    基于这些信息，智能体决定采取以下行动之一：
    -   **提交买单 (Buy Order)**: 以指定价格和数量买入资产。
    -   **提交卖单 (Sell Order)**: 以指定价格和数量卖出资产。
    -   **不交易 (Hold)**: 持有当前资产组合，不参与本期交易。

-   **资产估值**: 部分智能体维护对资产价值的主观估计（`asset_valuation`），该估值可能偏离基础价值，反映了市场中的异质性预期。

-   **动态演化**: 模拟引入了人口动态。新的交易者可能在模拟过程中加入市场（基于满意度的出生机制），同时也存在智能体退出（死亡）的可能。

#### 2.2.2. 实验资产 (Experimental Asset)

市场中交易的资产具有明确定义的特征：

-   **基础价值 (Fundamental Value)**: 资产的基础价值定义为剩余所有交易期的预期分红总额。公式为：
    $$ V_{fundamental}(t) = D \times (T - t) $$
    其中，$D$ 是每期固定分红额（`dividend_amount`），$T$ 是总交易期数（`trading_periods`），$t$ 是当前交易期。由于交易期有限，资产的基础价值随时间线性递减，最后一期结束后价值归零。

-   **分红机制**: 在每个交易期开始时，所有持有资产的智能体按其持有份额获得分红（`pay_dividend`）。这是智能体从资产中获取确定性收益的唯一途径。

-   **有限期限**: 资产的交易和分红仅持续固定的交易期数，期满后资产不再具有价值。这一设计使得基础价值可被精确计算，为观察价格偏离提供了清晰的基准。

#### 2.2.3. 市场微观结构：订单簿机制 (Order Book)

市场采用连续双向拍卖机制，通过订单簿进行交易撮合：

-   **订单簿 (Order Book)**: 维护两个队列：买单队列（`bids`）和卖单队列（`asks`）。每个订单包含价格、数量和提交者信息。
-   **订单撮合 (Order Matching)**: 在每个交易期的订单提交阶段结束后，系统执行撮合算法（`match_orders`）：
    1.  买单按价格从高到低排序，卖单按价格从低到高排序。
    2.  依次匹配最高买价和最低卖价。如果买价 ≥ 卖价，则成交，成交价格通常为两者的平均值。
    3.  根据双方订单数量确定成交量，更新双方的现金和资产持有。
    4.  部分成交的订单保留未成交部分，继续参与后续撮合。
    5.  重复此过程直至无法继续撮合。

-   **价格形成**: 当期市场价格（`current_price`）定义为本期所有成交价格的成交量加权平均值。如果本期无成交，则沿用上期价格或基础价值。

## 3. 核心机制与模拟流程

#### 3.1. 核心机制

1.  **基础价值锚定与价格偏离**: 
    -   资产的基础价值随时间确定性递减，为评估泡沫提供了客观标准。
    -   价格偏离率（`price_deviation`）定义为：
        $$ Deviation_t = \frac{P_{market}(t) - V_{fundamental}(t)}{V_{fundamental}(t)} \times 100\% $$
        其中，$P_{market}(t)$ 是市场价格。正偏离表示价格高于基础价值（泡沫），负偏离表示价格低于基础价值。

2.  **智能体LLM驱动的异质性决策**:
    -   不同于传统基于规则的代理模型，本模型中的智能体通过LLM进行决策，赋予其近似人类的推理能力。
    -   LLM接收详细的市场信息和个体状态，生成包含行动类型、价格和数量的结构化决策。
    -   智能体的交易策略和风险偏好在LLM的提示词中明确定义，确保决策行为的一致性和可解释性。

3.  **订单簿与流动性**:
    -   买卖价差（`bid_ask_spread`）定义为当期订单簿中最高买价和最低卖价之间的差距，是衡量市场流动性和交易成本的重要指标。
    -   价差越小，市场流动性越高，交易越容易执行；价差越大，表明市场参与者对价值的分歧越大。

4.  **分红的财富效应**:
    -   每期分红直接增加持有资产智能体的现金，提升其购买力，可能刺激后续的买入行为。
    -   分红也降低了资产的基础价值，理性智能体应相应调整其估值和交易策略。

#### 3.2. 模拟流程

模拟以离散的交易期（Period）进行，遵循 `simulator.run()` 方法中定义的循环。

1.  **初始化阶段**:
    -   根据 `config/asset_market_bubble_sim/simulation_config.yaml` 加载所有参数。
    -   创建地图、时间、人口、城镇等宏观环境对象。
    -   生成初始的交易者智能体群体，并构建社交网络。
    -   初始化资产市场（`initialize_assets`）：为每个智能体分配相等的初始现金和资产，并随机赋予交易策略和风险偏好。
    -   初始化订单簿和交易历史记录器。

2.  **迭代阶段 (按交易期循环)**:
    -   **交易期公告**: 输出当前交易期编号和资产的当前基础价值。
    -   **状态更新 (`update_state`)**:
        -   支付分红：所有持有资产的智能体按持有量获得现金分红。
        -   计算宏观指标：GDP、平均满意度。
        -   更新人口出生率（基于满意度）。
        -   政府决策（如果启用）。
    -   **执行行动 (`execute_actions`)**:
        -   **人口增长**: 根据出生率生成新的交易者，并整合进系统。
        -   **交易会话 (`conduct_trading_session`)**:
            1.  清空上期订单簿。
            2.  所有智能体并行提交订单（`resident_submit_order`）：
                -   调用LLM进行交易决策。
                -   将有效的买单和卖单分别加入订单簿。
            3.  执行订单撮合（`match_orders`）。
            4.  计算并记录当期市场价格。
        -   **生命周期管理**: 检查并移除"死亡"的智能体。
    -   **数据收集 (`collect_results`)**: 记录当前交易期的关键指标（详见第5节）。
    -   **结果保存**: 实时追加保存数据到CSV文件。
    -   **时间推进**: 进入下一交易期，循环继续，直至达到总交易期数。

3.  **结束与分析阶段**:
    -   模拟循环结束后，完整保存所有收集到的时间序列数据到CSV文件。
    -   调用可视化函数（`plot_asset_market_results`）生成一系列图表并保存。
    -   输出总模拟耗时。

## 4. 关键参数与可配置性

本模型具有高度的可配置性，研究者可通过修改位于 `config/asset_market_bubble_sim/` 目录下的 `simulation_config.yaml` 文件来调整实验条件。主要参数包括：

**资产市场参数**:
-   `initial_cash`: 每个智能体的初始现金禀赋。
-   `dividend_amount`: 资产每期支付的固定分红额。
-   `trading_periods`: 资产交易的总期数，也决定了资产的初始基础价值。

**智能体参数**:
-   `initial_population`: 初始市场参与者（交易者）的数量。
-   `birth_rate`: 基础人口年增长率，影响新交易者的加入速度。
-   `trading_strategy`（在代码中随机分配）: 智能体的交易策略类型，如"动量交易"、"价值投资"等。
-   `risk_preference`（在代码中随机分配）: 智能体的风险偏好，如"风险厌恶"、"风险中性"、"风险偏好"。

**模拟控制参数**:
-   `start_year` / `total_years`: 模拟的起止时间范围（映射到交易期）。
-   `map_width` / `map_height`: 虚拟地图尺寸（用于空间关系建模）。

**社会结构参数**:
-   `towns_data_path`: 城镇配置数据，影响智能体的初始分布和社交网络结构。

通过系统性地调整这些参数（如改变初始现金与资产比例、分红额、交易策略分布等），可以研究不同市场条件对泡沫形成的影响，验证理论假说。

## 5. 数据收集与评估指标

为了全面评估资产市场泡沫的形成、演化和市场微观行为，我们在每个交易期收集了以下一系列关键指标：

| 指标名称 (代码内) | 指标含义与计算方式 | 研究意义 |
| :--- | :--- | :--- |
| `asset_price` | 当期市场价格，为当期所有成交的成交量加权平均价格。 | 核心观察变量，反映市场对资产价值的即时评估。 |
| `trading_volume` | 当期总成交量（份数）。 | 衡量市场活跃度和流动性，高交易量可能伴随价格剧烈波动。 |
| `average_cash` | 所有智能体的平均现金持有量。 | 反映市场整体的流动性和购买力，现金减少可能预示泡沫后期。 |
| `average_assets` | 所有智能体的平均资产持有量。 | 跟踪资产分布的变化，可分析资产向特定类型交易者集中的现象。 |
| `price_deviation` | 价格偏离率，计算为 $(P_{market} - V_{fundamental}) / V_{fundamental} \times 100\%$。 | **核心泡沫指标**。正值且持续增大表明泡沫形成；峰值及后续下降可能标志泡沫崩溃。 |
| `bid_ask_spread` | 买卖价差，为当期订单簿中最高买价与最低卖价之差。 | 衡量市场流动性和交易成本，价差扩大可能预示市场不确定性增加。 |
| `population` | 当期市场参与者总数。 | 跟踪市场规模变化，可与其它指标关联分析规模效应和拥挤交易。 |

## 6. 实验结果的体现与分析

模拟的输出结果为深入分析泡沫形成机制提供了丰富的数据基础：

1.  **原始数据**: 所有在模拟过程中按交易期收集的指标数据，以时间序列的形式保存在 `data/` 目录下的CSV文件中（文件名格式为 `running_data_{timestamp}_pid{pid}.csv`）。该数据集支持：
    -   时间序列分析：识别泡沫的形成期、膨胀期、峰值期和崩溃期。
    -   统计检验：验证价格是否显著偏离基础价值。
    -   相关性分析：探索价格偏离与交易量、买卖价差等指标的关联。

2.  **可视化图表**: 模拟自动生成一系列图表，保存在 `experiment_dataset/asset_market_bubble_sim/plots/` 目录下，包括：
    -   **资产价格变化图**: 绘制 `asset_price` 的时间序列，直观展示价格的波动、趋势和潜在的泡沫轨迹。可叠加基础价值曲线作为对比。
    -   **交易量变化图**: 绘制 `trading_volume` 的时间序列，分析市场活跃度的演变。通常在泡沫形成初期交易量激增，后期可能因谨慎情绪而下降。
    -   **价格偏离基础价值图**: 绘制 `price_deviation` 的时间序列，并标注零线（无偏离）。该图是识别和量化泡沫的核心工具。
    -   **市场流动性指标图**: 绘制 `bid_ask_spread` 的时间序列，评估市场微观结构的健康程度。

3.  **典型分析视角**:
    -   **泡沫识别**: 通过观察价格偏离率是否持续为正且增大，判断泡沫的存在。
    -   **泡沫周期划分**: 根据价格偏离的变化率，将模拟分为形成期（偏离从零开始增加）、膨胀期（偏离加速上升）、峰值期（偏离达到最大）和崩溃期（偏离快速回落，价格向基础价值收敛）。
    -   **行为模式分析**: 结合智能体的交易策略分布和LLM决策日志，分析不同类型交易者（如动量交易者 vs 价值投资者）在泡沫各阶段的行为差异。
    -   **政策干预实验**: 如果启用政府模块，可模拟监管政策（如交易税、涨跌幅限制）对泡沫的抑制效果。

通过对这些结果的综合分析，研究者可以：
-   **验证理论假说**: 如Smith, Suchanek和Williams（1988）提出的"不确定性和投机"是泡沫形成的关键因素。
-   **识别关键驱动因素**: 通过参数敏感性分析，确定哪些因素（如智能体异质性程度、初始资金量、分红水平）对泡沫的幅度和持续时间影响最大。
-   **探索新问题**: 如社交网络结构对泡沫传播的影响，或不同LLM提示策略对智能体理性程度的影响。

## 7. 结论与展望

本模拟模型提供了一个灵活且可扩展的平台，用于研究资产市场泡沫的内生形成机制。通过结合多智能体系统、行为金融学原理和大型语言模型的推理能力，该模型能够复现现实市场中观察到的泡沫现象，并为理解其背后的微观行为基础提供洞见。未来的研究可以扩展模型，纳入更复杂的市场微观结构（如做市商、卖空机制）、信息不对称、以及更丰富的智能体学习和适应机制，以进一步提升模型的现实性和解释力。
